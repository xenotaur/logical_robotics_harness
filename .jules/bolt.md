## 2024-07-05 - Markdown Parsing String Iteration
**Learning:** In Python, iterating line-by-line using a `while` loop and `.find('\n')` to search for a delimiter like `---` can be extremely slow on large files without the delimiter (worst-case scenario), resulting in millions of unnecessary string slices. Standard regex `re.MULTILINE` is faster but can still backtrack heavily.
**Action:** Use `str.find('---', start_index)` to jump directly to candidate matches, then use `rfind('\n')` and `find('\n')` around that index to isolate the specific line for exact checking. This drastically reduces the number of string allocations and loops, dropping worst-case execution time from O(N lines) down to a fraction of a second.
## 2024-07-07 - File Traversal With Pathlib rglob
**Learning:** `pathlib.Path.rglob()` always traverses the full subtree, yielding all items, and lacks an inherent mechanism to prune or skip directories *during* the tree walk (unlike `os.walk` where one can mutate the `dirnames` list). Checking if ignored patterns exist via `any(part in ignores for part in path.parts)` inside an `rglob` loop skips processing, but not traversal, resulting in massive worst-case slowdowns if heavy node_modules or large virtualenvs are present in the repo.
**Action:** Always prefer `os.walk` over `rglob` when working with potentially heavy and unconstrained code bases, leveraging `dirnames[:] = [d for d in dirnames if not skip(d)]` to prune the graph traversal early.
