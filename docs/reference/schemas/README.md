# Schema Reference

Schema reference documents LRH file formats, frontmatter fields, control-plane structure, and validation rules.

## What belongs here

- Stable Markdown/frontmatter field definitions.
- Control-plane directory and file-format rules for reusable LRH projects.
- Validation semantics that users need to satisfy intentionally.
- Compatibility notes for format evolution.

## What does not belong here

- Active project plans, status, evidence, or decisions copied from [`../../../project/`](../../../project/).
- Tutorials that teach schemas through a complete example project.
- How-to recipes for repairing a specific repository.
- Design discussions that do not define current accepted behavior.

## How to decide whether to add content here

Add content here when the content defines a stable contract that LRH code, users, and project control files must agree on. If a schema rule is still an active design proposal or project decision, link to the authoritative artifact under [`../../../project/`](../../../project/) instead of treating it as settled reference.

## Currently relevant docs

No schema reference pages have been migrated yet. Future PRs should add pages only as the corresponding behavior is stable enough to maintain as reference.
