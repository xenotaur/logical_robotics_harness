# Tutorials

Tutorials are complete, guided learning paths for people who are new to LRH or to a major LRH workflow.

## What belongs here

- End-to-end walkthroughs that teach by doing.
- Learning paths with a clear starting point, sequence, and expected result.
- Small example projects or toy scenarios that make LRH concepts concrete.
- Beginner-friendly introductions that intentionally avoid exhaustive reference detail.

## What does not belong here

- One-off operational recipes for people who already know what they need to do.
- Exact command catalogs, schema listings, or file-format specifications.
- Rationale-only design essays with no guided exercise.
- Authoritative project state copied from [`../../project/`](../../project/).

## How to decide whether to add content here

Add content here when the reader's main goal is learning a workflow from start to finish. If the reader is trying to complete a specific task, use [how-to guides](../how-to/README.md). If they need stable facts or exact syntax, use [reference](../reference/README.md). If they need to understand why LRH works this way, use [explanations](../explanations/README.md).
