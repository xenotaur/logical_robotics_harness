# Serve Interface Steward — Memory

Reviewed institutional knowledge for this assistant role. Memory is **advisory**:
accepted memory may guide behavior but does not participate in LRH precedence
and cannot override principles, goals, roadmap, focus, approved design,
workstream scope, or work-item acceptance. Its relationship to
`project/memory/decision_log.md` and `project/memory/decisions/` is one of
non-precedence advisory guidance, not a competing decision record.

## Lifecycle buckets

- `proposed/` — `AMEM-*` items the assistant proposed (it holds only
  `memory:propose`). Awaiting human review.
- `accepted/` — items a human reviewer promoted. Promotion requires a human
  reviewer, a review timestamp, retained provenance, and validation.
- `retired/` — items no longer in force, retained for history.

## The assistant may never

Accept its own memory, modify its policy, remove mandatory escalations, alter
its capability ceiling, rewrite acceptance criteria, or retire inconvenient
memory without review.

## Item shape

Each proposed item records: assistant ID, source run or workstream, evidence
links, scope, creation timestamp, confidence, and the proposed lesson. See
`proposed/AMEM-EXAMPLE-SERVE-LATENCY.md` for a worked example.
