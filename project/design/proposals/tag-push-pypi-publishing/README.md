# Tag-push PyPI publishing proposal set

This proposal set defines Option D release design for publishing the
safe-default `lrh` distribution through tag-push GitHub Actions using
PyPI Trusted Publishing.

## Status

`accepted`

## Documents

1. [`00_proposal.md`](00_proposal.md)
   — umbrella proposal covering motivation, safe-default semantics,
   Trusted Publishing posture, workflow shape, TestPyPI rehearsal,
   release smoke/evidence requirements, tradeoffs, risks, and phased
   follow-up implementation guidance.

## Scope notes

This accepted proposal set records canonical release direction only and
does not implement release workflows, packaging metadata edits, package
split changes, or publish operations.
