# Prompt execution search and match implementation evidence

This evidence record supports adopting
`PROP-PROMPT-EXECUTION-SEARCH-AND-MATCH` as implemented for its first
command-surface slice.

## Evidence

- `src/lrh/prompt_workflow_records.py` defines the package-owned
  `ExecutionRecord` model plus execution-record parsing and loading from
  `project/executions/`.
- `src/lrh/prompt_workflow_queries.py` provides the reusable exact
  prompt-ID lookup used by prompt workflow commands.
- `src/lrh/prompt_workflow_match.py` implements exact prompt-file matching
  and the `lrh match executions` command surface.
- `src/lrh/prompt_workflow_search.py` implements exploratory deterministic
  substring search and the `lrh search executions` command surface.
- `src/lrh/cli/main.py` dispatches installed `lrh match` and `lrh search`
  commands to the package-owned implementations.
- `tests/assist_tests/prompt_workflow_match_test.py` and
  `tests/assist_tests/prompt_workflow_search_test.py` cover match and search
  behavior, including exact matching, no-match behavior, filtering, ordering,
  and command formatting paths.
- `project/executions/README.md` documents `lrh prompt check-execution` as
  authoritative, `lrh match executions` as exact prompt-file convenience, and
  `lrh search executions` as exploratory context only.

## Interpretation

The implemented command surface preserves the proposal's evidence hierarchy:
structured exact prompt-ID lookup remains authoritative for soft idempotence,
`lrh match executions` delegates to exact lookup, and `lrh search executions`
is explicitly exploratory and non-authoritative for rerun decisions.
