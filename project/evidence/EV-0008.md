# Safe-default agentic packaging boundary adoption evidence

This evidence record supports adopting
`PROP-SAFE-DEFAULT-AGENTIC-EXTRA-PACKAGING` as partially implemented.

## Evidence

- `project/design/design.md` records `lrh` as the safe-default user-facing
  distribution and states future autonomous capability must stay explicit
  through `lrh[agentic]` and/or `lrh-agentic` when that package exists.
- `project/design/architecture.md` records the same safe-default non-agentic
  distribution boundary and describes it as packaging/governance rather than a
  security sandbox.
- `project/design/repository_spec.md` records the default `lrh` package as
  non-agentic and future autonomous capability as opt-in.
- `docs/release.md` documents the default `lrh` install as non-agentic and
  warns not to present future `lrh[agentic]` modes as current behavior before
  the extra/package behavior is implemented.
- `pyproject.toml` exposes the `lrh` console script and currently has no
  `agentic` optional dependency, matching the safe-default side of the adopted
  boundary.

## Interpretation

The safe-default packaging/governance decision has been adopted into canonical
project design documents and current package metadata does not install an
agentic extra. The implementation remains partial because the optional
`lrh[agentic]` extra, separate `lrh-agentic` distribution, and integrated
`lrh agentic` missing-package guidance are still deferred until concrete
autonomous execution implementation exists.
